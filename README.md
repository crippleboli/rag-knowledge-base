# 基于多源异构文档解析与多路召回策略的检索增强生成系统

## 1. 项目概述

本项目是一套基于 RAG（检索增强生成）架构的企业级私有知识库智能问答系统，专注解决企业内部知识管理、智能客服、专业咨询等场景的 AI 落地难题。系统工程化代码规模达 5500+ 行，完整实现了从文档接入、知识库构建、智能检索到答案生成的全链路生产级能力。

### 核心定位与价值
* 核心定位：全链路、高鲁棒性、可扩展的企业级智能问答系统。
* 安全合规：支持私有知识安全管理与私有化部署。
* 精准降噪：从根源降低大模型事实幻觉风险，确保应答内容基于权威事实。
* 多模态接入：支持多格式文档自动解析与结构化入库。
* 高级策略融合：集成多路检索、混合向量、重排序（Rerank）、联网补充等策略。

---

## 2. 技术基础

RAG（Retrieval-Augmented Generation）采用“先检索、后生成”的架构，先从企业内部私有知识库精准匹配相关资料，再由大模型依托真实文档素材生成答案。

### 关键模型应用
* Embedding 向量化：将文本转换为向量表示，支持稠密与稀疏向量，为高效相似度检索提供基础。
* Rerank 重排序：对召回结果做精细化排序，过滤低相关片段，显著提升上下文质量。
* LLM 大模型生成：基于检索增强后的高质量上下文，结合用户问题，生成可信、精准的最终回答。
* 视觉模型：针对用户上传的图片、截图等内容完成图像解析，支撑图文混合场景。

## 3. 项目流程与技术选型

### 3.1 项目流程梳理

#### 1. 知识库构建流水线（离线数据处理）
实现原始文档的结构化接入与向量化资产沉淀。通过任务分发机制适配多格式源数据，结合结构化解析、图片理解与智能分块技术，完成文档标准化处理。
核心链路：原始文档 -> 任务分发 -> PDF结构化解析 -> 多模态图片理解 -> 智能文档切片 -> 主体识别与标签提取 -> 混合向量化 -> 数据持久化。

#### 2. 智能检索与生成流水线（在线问答服务）
面向用户查询实现低延迟、高可信的智能问答交付。通过意图理解提升查询精准度，采用多策略召回扩大知识覆盖范围，经排序优化后由 LLM生成答案，并以流式输出。
核心链路：用户查询 -> 意图识别与改写 -> 多路召回（向量搜索/HyDE/网络搜索） -> 结果融合与粗排 -> 重排序 -> LLM生成答案 -> SSE流式输出。

### 3.2 技术选型与理由

项目选择 LangGraph 图结构工作流作为核心编排方案，而不采用黑盒式原生 Agent。

主要考量：
* 流程完全可控：LangGraph 通过显式定义节点和执行路径，杜绝了 Agent 自主决策带来的偏离与不可控问题，确保生产环境稳定。
* 执行路径可追溯：图结构能清晰记录每一步执行节点与状态变化，满足企业对问答溯源和故障定位的刚性需求。
* 状态管理稳定：具备全局状态共享能力，可在多轮问答、多路检索中稳定传递上下文。

## 4. 注意事项

### 代理敏感性问题
MinerU 云端接口对网络代理极其敏感。在本地开启代理软件（如 Clash、v2ray 等）时运行网页接口服务，上传 PDF 会高概率触发 401 Unauthorized 错误。

### 原因
FastAPI/Uvicorn 服务启动时，Python 进程会自动捕获系统当前的代理环境变量并缓存在内存中。即使在网页运行期间手动关闭代理软件，Python 进程内的变量也不会自动刷新，请求依然会被污染。

### 解决方案
在进行接口调用或服务启动前，请选择以下方案之一：
1. 代码层防错：在网络请求中，统一使用 `requests.Session(trust_env=False)` 强制无视系统环境代理，直接直连外网。
2. 彻底清理进程：若需关闭代理，关闭后请在终端执行 `taskkill /f /im python.exe` 强行杀死残留的 Python 后台进程，再重新启动 `import_server.py`。
